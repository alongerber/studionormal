# חצינורמלי — סטודיו תוכן (v2)

כלי ליצירה וליטוש של תוכן בארבעה פורמטים + בנק רגעים + לוג ביצועים + בודק קנון אוטומטי.

---

## מה חדש ב-v2

### 🧠 בנק רגעים
כל רגע קטן שעולה בראש — משפט אחד, סיטואציה ויזואלית, פרט מעצבן — נשמר. בלחיצה, המודל הופך אותו לבריף מובנה ויוצר עבודה בכל אחד מ-3 המצבים.

### 📚 קונטקסט מוזרק אוטומטית
טוען את `canon.md`, `timeline.md`, `topics_map.md` (ועוד) פעם אחת, ומכאן המודל מכיר את הקנון בכל בקשה. לא מתבלבל בין ארבל לכלב של האקס, לא ממציא גיל חדש לאמא.

### 🔍 בודק קנון (Canon Linter)
אחרי שמייצרים טיוטה — בדיקה אוטומטית ברקע של סתירות עם הקנון. מציג רשימת בעיות (סתירות / המצאות / הפרות איסור / חוטים חדשים) עם ציטוט מדויק והצעת תיקון.

### 💭 מצב תגובה חיצונית
מצב רביעי: מדביקים פוסט של עמוד חיצוני + קטגוריה (7 סוגים: יועצים/סופרים/מותגים/מקומיים/חינוך/אוכל/דייטינג), מקבלים 4 תגובות בסגנון הדמות שעומדות לבד בלי הקשר מהעמוד שלך.

### 🔁 המרה בין מצבים
פוסט מאושר → כפתור אחד → סרטון / וואטסאפ. או להפך. אותו סיפור, מדיום חדש.

### 📈 לוג ביצועים
אחרי שפוסט מאושר במלואו, לחיצה על 📈 בכרטיס פותחת מסך שמירת נתונים (צפיות, שיתופים, עוקבים, סוג חשיפה). מנתח דפוסים אוטומטית כשיש 3+ רשומות. ייצוא CSV.

### ⚡ הפרדת מודלים חכמה
משימות קלות (angles, hooks, alternatives, fix, score) → **Sonnet 4.6** (זול פי 8).
משימות יצירתיות (draft, callbacks, canon check, reply) → המודל שבחרת (ברירת מחדל Opus 4.7).

### 🧠 זיכרון בין מצבים
כל מצב זוכר את העבודה האחרונה שהיית בה. החלפת מצב לא מאבדת מידע.

## 🚀 העלאה ל-Netlify

### שיטה 1: Drag & Drop (הכי מהיר)
1. לך ל-[netlify.com/drop](https://app.netlify.com/drop)
2. גרור את התיקייה `hatzi-studio` כולה (index.html, app.js, styles.css, prompts.js, storage.js)
3. תוך 10 שניות יש URL עובד

### שיטה 2: Netlify CLI
```bash
cd hatzi-studio
npm install -g netlify-cli
netlify deploy --prod --dir=.
```

### שיטה 3: ריצה מקומית (לבדיקה)
```bash
cd hatzi-studio
python3 -m http.server 8080
# פתח http://localhost:8080
```

⚠️ **חשוב:** הכלי לא יעבוד מקובץ `file://` — צריך HTTP server. זו מגבלה של IndexedDB בדפדפן.

## ⚙ הגדרה ראשונית

1. פתח את ה-URL של Netlify
2. לחץ על גלגל ההגדרות (⚙) → הכנס API key של Anthropic
3. לחץ על 📚 → העלה את קבצי הקנון שלך (canon.md, timeline.md, topics_map.md — אפשר כולם ביחד)
4. מודל ברירת מחדל: **Opus 4.7**. Sonnet 4.6 ממילא רץ אוטומטית על המשימות הקלות.

## 📖 זרימות עבודה

### 🧠 מרגע לפוסט:
1. חושב "הילד הסתכל היום על הסופר פארם" → 🧠 רגעים → + שמור רגע
2. בסוף השבוע: 🧠 → "+ הפוך לעבודה" → בוחר מצב (פוסט/סרטון/וואטסאפ)
3. ה-AI מחזיר בריף מובנה עם זווית מוצעת
4. ממשיך כרגיל (draft → polish → export)

### 📝 יצירת פוסט:
1. **ארכיון** → **+ חדש** → נכנסים ל**ייצור**
2. כותבים בריף
3. **4 זוויות** → בוחרים זווית
4. **ייצר פוסט**
5. **בדיקת קנון רצה ברקע אוטומטית** — אם יש סתירות, toast
6. **העבר לליטוש** → עוברים פסקה-פסקה, 🔍 לבדיקת קנון ידנית
7. **ייצוא**
8. אם רוצים — כפתור "🔁 הפך ל-סרטון/וואטסאפ" בייצוא

### 💭 יצירת תגובה חיצונית:
1. מצב **💭 תגובה** → חדש
2. מדביקים את הפוסט החיצוני, שם העמוד, בוחרים קטגוריה
3. **ייצר 4 תגובות** → מקבלים 4 חלופות
4. מעבירים לליטוש — כל תגובה היא "שורה" (בלוק)
5. בוחרים את הטובה, מאשרים, מייצאים

### 📈 מעקב ביצועים:
1. אחרי שפוסט מאושר במלואו — בארכיון יופיע כפתור ➕ על הכרטיס
2. אחרי הפרסום — 48 שעות → חזרה, לחיצה על 📈 → מילוי נתונים
3. מעל 3 רשומות — 📈 בכותרת → "נתח דפוסים" → המודל מחזיר תובנות

## ⚠ נקודות לזכור

1. **הקנון הוא האמת.** מה שכתוב ב-`canon.md` גובר על כל דבר אחר. לפני פרסום פוסט חדש — לעדכן את הקנון.
2. **קבצי הקונטקסט נטענים פעם אחת ונשמרים.** לא צריך להעלות שוב כל פעם. כשהקנון משתנה — העלה מחדש.
3. **הפרדת מודלים חוסכת כסף.** משימות קלות רצות תמיד על Sonnet. ה-selector למעלה רק משפיע על משימות יצירתיות.
4. **המצב הנוכחי שמור תמיד.** גם במעבר בין מצבים — כל מצב זוכר את העבודה האחרונה.
5. **הגיבוי JSON כולל רגעים + הגדרות קונטקסט** (לא API key). בבטחה לעלות ל-Google Drive.

## 🐛 בעיות נפוצות

**"שגיאת API: invalid_request"** — ה-key לא תקף. בדוק ב-console.anthropic.com שהוא פעיל.

**"הקנון לא מוזרק"** — פתח 📚, בדוק שהקבצים טעונים. לחץ "👁 ראה איך זה נראה ב-prompt" כדי לאמת.

**"canon check לא רץ"** — הוא רץ אוטומטית רק אם יש קבצי קונטקסט טעונים. אחרת משתמש ב-🔍 ידנית.

**"המרת מצב נתנה פלט משונה"** — ודא שהטקסט מאושר לפני ההמרה. המודל עובד על מאושרות, ואם אין — על הכל.
